The dataset that I gonna investigate is that of the "Tmdb dataset". This data set contains information about 10,000 movies collected from The Movie Database (TMDb), including user ratings and revenue. It has various columns including, popularity,budget,revenue,original_title,keywords,overview,runtime,genres,production_companies,release_date among many others. Looking at the dataset, some of the questions which would be interesting to answer are as follows:

  1. Which genres are most popular from year to year?
  2. What kinds of properties are associated with movies that have high revenues?
  3. Given a movie, can I return the Movie's tagline?

Going forward from here, I shall list down my steps of cleaning the data, and applying various operations in order to arrive at a result for the above questions.

## function to return the tagline of the film selected

def tagline_film(film_name):
    pos=0
    for data in film_data['original_title']:
        if(data==film_name):
            return film_data['tagline'][pos]
        pos+=1

#generating a frequency series for the list of genres in a particular year
def generate_frequency_of_movies(genres_array,release_year):
    word_count={}
    for words in genres_array:
        if(words in word_count):
            word_count[words]+=1
        else:
            word_count[words]=1
            
    genres = pd.Series(word_count)
    display_bar_graph_for_genres_series(genres,release_year)

LIMITATIONS:

  1. Some obvious problem that I encounter in the dataset is that of a common currency to state the values of revenue and budget. If films are made in different countries with different budget, we need to have a common scale in order to be able to analyze them properly.

  2. The popularity of the movie is a very vague term. It does not necessarily mean that if the film was viewed by a large audience that it was more popular. Rather than that, the weekly views of the movie is a better option, as a consistent viewing every week would state more popularity than a movie which drastically dropped in views after the first week.

  3. There are a few null values in the dataset, and ignoring them completely in my analysis could have resulted in a slightly skewed graphs, and incorrect predictions. An efficient way of handling null values needs to be developed.

  4. There are a slighly complex and much more accurate models to analyse data, and the readers are advised to definitely look into them, as mine is a rather primitive form of analysis of a beginner who has just started out.
